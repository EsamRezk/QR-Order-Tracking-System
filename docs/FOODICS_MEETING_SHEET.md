# 📄 ورقة اجتماع فوديكس — كبة زون

> اطبعها أو افتحها في الاجتماع. كل سطر فيه **افتراضنا الحالي** — يا تأكّده يا يصلّحه.
> بعد الاجتماع: نعدّل ملف واحد (`foodics-status.ts`) + نملأ branch IDs + التوكن.

---

## ⭐ أولوية قصوى (لو مرجعش، التكامل مايشتغلش)

### 1. هل نقدر نكتب حالة الطلب عبر API؟
- [ ] فوديكس بيسمح نحدّث **حالة التوصيل** من برّه (API)؟ نعم / لا
- [ ] الـ endpoint الصح؟  (افتراضنا: `PUT /orders/{id}`)
- [ ] الـ scope المطلوب على التوكن للكتابة؟  (افتراضنا: `orders.limited.deliver`)

### 2. قيم delivery_status (الأرقام الدقيقة)
| المرحلة | افتراضنا | الصح |
|---|---|---|
| Sent to Kitchen | 1 | ____ |
| Ready | 2 | ____ |
| Delivered / Picked up | 5 | ____ |
| Cancelled | 6 | ____ |
| Assigned / Enroute (نتجاهلها) | 3 / 4 | ____ |

- [ ] اسم الحقل فعلاً `delivery_status`؟  أرقام ولا نصوص؟

### 3. الـ Webhook
- [ ] أسماء الأحداث الصح؟  (افتراضنا: `order.delivery.created` + `order.delivery.updated`)
- [ ] الـ payload بيحمل الطلب **كامل** ولا نجيبه بـ API؟  (افتراضنا: كامل)
- [ ] **مين يسجّل الـ webhook؟** انت من اللوحة ولا support@foodics.com؟
      الرابط: `https://<PROJECT_REF>.supabase.co/functions/v1/foodics-webhook`
- [ ] فيه توقيع/سر HMAC؟  (افتراضنا: لا)

### 4. ربط الفروع
- [ ] **Foodics branch ID (UUID)** لكل فرع:
  - عرقه (Erqaa-01): ________________________
  - ظهرة لبن (Laban-02): ________________________
  - الملقا (AlMalqa-03): ________________________

## 🟡 أولوية متوسطة

### 5. الكتابة (عندنا → فوديكس)
- [ ] زر "جاهز" نرسل إيه؟  (افتراضنا: `{ delivery_status: 2 }`)
- [ ] زر "تم التسليم" نرسل إيه؟  (افتراضنا: `{ delivery_status: 5, driver_collected_at: "<UTC>" }`)
- [ ] صيغة الوقت؟  (افتراضنا: `YYYY-MM-DD HH:MM:SS` UTC)

### 6. قيم status للإلغاء
- [ ] Declined = ؟  (افتراضنا: 3)   |   Void = ؟  (افتراضنا: 7)

### 7. كشف تطبيق التوصيل (للوجو/اللون على الكروت)
- [ ] فيه `app_id` فريد لكل تطبيق (كيتا/هنقرستيشن/جاهز...)؟
- [ ] ولا الاسم في حقل نصي (aggregator / channel / reference)؟
- [ ] **اطلب عيّنة JSON حقيقية لطلب توصيل من تطبيق فعلي** (مش كاشير يدوي)

## 🟢 تأكيدات سريعة

- [ ] التوكن الإنتاجي (قراءة + كتابة حالة التوصيل)
- [ ] `api_base_url` = `https://api.foodics.com/v5`؟
- [ ] الـ rate limit للـ API؟
- [ ] زمن وصول الـ webhook المتوقع؟  (نستهدف ≤ 5 ثواني)

## الفلو اللي بنأكّده معاهم
```
طلب delivery → الكاشير "إرسال للمطبخ"
   → يظهر عندنا "قيد التحضير"        (قراءة من فوديكس)
   → الشيف "جاهز"    → "جاهز" عندنا + عند فوديكس   (كتابة)
   → الشيف "تم التسليم" → "تم التسليم" عندنا + فوديكس (كتابة)
ولو الكاشير غيّر الحالة من فوديكس → تتحدّث عندنا تلقائياً (قراءة)
```
