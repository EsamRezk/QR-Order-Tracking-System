# تصميم شاشة المطبخ (`/kitchen`)

> **التاريخ:** 2026-03-27

---

## الهدف

إضافة شاشة مطبخ تعرض الطلبات "قيد التحضير" بتصميم Grid مع زر "جاهز" لكل طلب، بدلاً من الحاجة لمسح QR مرة ثانية.

---

## المسار والصلاحيات

- **Route:** `/kitchen?branch=CODE`
- **الأدوار المسموحة:** `user` + `admin`
- إذا لم يُحدد `?branch=` → يعرض `BranchSelect`
- اليوزر يتوجه تلقائياً لشاشة المطبخ حسب `route` المحفوظ في حسابه

## البطاقة (Kitchen Order Card)

- نفس بيانات `OrderCard` الحالي: رقم الطلب، قناة التوصيل، وقت الانتظار
- نفس التصميم العام (ألوان، خطوط، حدود) للتناسق مع باقي النظام
- **إضافة:** زر "جاهز" أخضر (`--kz-green`) أسفل البطاقة

---

## Grid Layout

- `display: grid` مع `auto-fill` و `minmax(280px, 1fr)`
- يتكيف تلقائياً مع حجم الشاشة (3-4 أعمدة على شاشة كبيرة، عمود واحد على موبايل)

---

## رسالة التأكيد (Modal)

- Modal في نص الشاشة مع خلفية شفافة داكنة
- النص: "هل تريد تحويل الطلب **KZ-1001** إلى جاهز؟"
- زرين: **تأكيد** (أخضر) + **إلغاء** (رمادي)

---

## Realtime

- نفس `useOrders` hook الموجود — يجلب طلبات `preparing` فقط
- لما يتحول طلب لـ `ready` يختفي من الشاشة بـ fade animation

## منطق زر "جاهز"

1. يضغط الزر → يظهر Modal التأكيد
2. يضغط "تأكيد" → `UPDATE orders SET status='ready', ready_at=now() WHERE id=...`
3. يضيف `INSERT scan_logs` بـ `scan_type = 'second_scan'`
4. الطلب يختفي من الشاشة عبر Realtime subscription
